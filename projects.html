AI project: Genetic Algorithms and the Eight Queen Puzzle

GitHub Link: https://github.com/gsampang/8-Queens-AI-Program-Showcase

Project Description:

The 8-Queens AI Program is a smaller scale project I worked on by myself as an assignment for one of my most challenging classes - AI. As such it is easy to run the code, just download the code from the main file and run with anything that can run C++.

The program solves the Eight Queens Puzzle by applying the concept of genetic algorithms.

The Eight Queens Puzzle is simple - just put 8 queens on a regular 8x8 chessboard with none of them attacking each other.

Genetic algorithms is a style of programming that simulates survival of the fittest, and in this case the fittest is the solution to the problem. It is a relatively easy to understand AI program as it is inspired by a concept many are familiar with from school - survival of the fittest.

The purpose of this project was to put into practice the concept of genetic algorithms. The program was developed in C++ as it is the language I was the most familiar with at the time.

Capstone Project: Climate Data Tool

GitHub Link: https://github.com/gsampang/ClimateDataCapstone2022

Project Description:

The Climate Data program is a much larger scale project I worked on as part of a team for the duration of 2 terms. Because of this, it is more complicated to set up the program. There are many prerequisites that need to be installed before the program can be run. These are outlined in the project readme.

The program’s use is relatively simple. From a database it takes climate related data from around the US and displays it in a way useful to our sponsor. Its main use is to create a graph based on a number of different options.

My main contribution to this project is to help implement the save state. I worked on this directly with a couple other group members. While the code has been some of the most challenging I’ve worked with, the real challenge was in coordinating with the other group members.

Our team was working remotely and for the most part asynchronously, which made communication much harder. This meant preventing redundant work staying in sync was a real challenge. To face this challenge I scheduled separate meetings where we discussed the code we had worked on in detail and found a way to merge our code. Setting up this process greatly helped as the term went on and the program got more and more complex.
